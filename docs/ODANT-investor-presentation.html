Проблема рынка

Организации собирают цифровой контур из разрозненных продуктов: мессенджеры, диски, CRM, офисные редакторы, корпоративный ИИ. Каждый инструмент хранит свой фрагмент истории — без единого следа деятельности, аудита и контекста для принятия решений.

Боль Типичное решение Последствия
Коммуникации Slack, Teams, Telegram Сообщения оторваны от документов и процессов
Документы SharePoint, Google Drive Нет связи с перепиской и решениями
Процессы CRM, ERP, no-code Дорогая интеграция, vendor lock-in
ИИ Внешние copilot / RAG Данные уходят в SaaS, слабый контроль
Регуляторика Западные облака Сложно развернуть on-premise с полным суверенитетом
Итог: высокая стоимость владения, фрагментированная память организации, медленная адаптация IT-системы под реальные практики подразделений.

Решение ODANT

ODANT (WORK) — файлово-событийная платформа: любая рабочая активность материализуется как файл в иерархии хранилищ организации. Сохранение файла — событие с историей, журналом и возможностью ИИ-анализа.

Организация └── Подразделения (хранилища $group, $structure…) └── Артефакты: сообщения, документы, письма, звонки, ответы ИИ └── history + logs + RAG → «память организации»

Ключевые отличия архитектуры

Фрактальная FS Папка = объект с типом. URL = путь к объекту. API = query-параметр. Отдельные БД и REST-контроллеры не проектируются вручную.
Наследование Поведение и UI наследуются слоями: тип → структура → подразделение. Кастомизация без пересборки ядра.
Чат ≠ отдельная БД Чат — представление над журналом событий. Сообщение = message.txt, ответ ИИ = response.md. Единый след для аудита и RAG.
Self-hosted Развёртывание on-premise, white-label PaaS, полный контроль данных — ответ на запрос enterprise и регулируемых отраслей.

Дорожная карта

  • Сейчас: ядро, ODA, чат, звонки, документы, RAG, публичное демо
  • Ближайшее: единый клиентский UI (~/client/pages/), стабилизация архитектуры
  • Vision: автопрограммирование — ИИ-агент администратора адаптирует систему по команде в чате

Инвестиционный тезис

  • Папка = объект = API = UI — радикально снижает стоимость разработки и внедрения
  • Self-hosted и white-label — ниша суверенитета данных и регулируемых отраслей
  • ИИ встроен в модель данных, а не прикручен сбоку
  • Технологический ров: фрактальная FS с наследуемым поведением сложно быстро воспроизвести
