# $product — продукт витрины

## 1. Что это

Тип файла `.product` — позиция каталога витрины (`MARKET`). JSON с описанием товара (тарифа): имя, цена, состав, форма заказа.

## 2. Зачем это нужно

Товар витрины — это **данные**, а не типизатор. Раньше тарифы PaaS были дочерними `$class` с `$public` — это смешивало тип и контент. Теперь товар — обычный файл в `work/product/` категории, создаваемый по роли.

## 3. Как это работает

1. Продукт создаётся в зоне роли (`get_storage`): USER → `meta_folder/work/product/*.product`.
2. Каждый `save_file` перезаписывает текущий файл целиком → снимок в `history/` + запись в `data.logs` класса (канон §1.6).
3. Витрина (`site/main`) получает список продуктов через `~//product` и рисует карточки из `DATA` файла.
4. Заказ сохраняет `{uid}.bid` на ближайший `$class` категории (не на файл продукта).

## 4. Из чего это состоит

- `class.js` — схема `FIELDS` продукта (label, icon, price, priceHint, includes, orderForm, status)
- `template.product` — пустой JSON для `create`

## 5. В каком это состоянии

- ✅ Схема и шаблон
- 🔧 Витрина MARKET/PAAS на `.product`
- ❌ Preview / `on_save` (не нужны для MVP)
