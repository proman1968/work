# $bid — заявка

## 1. Что это

Тип файла `.bid` — базовая заявка на покупку позиции витрины (`MARKET`). JSON с покупателем, снимком товара и ответами формы.

## 2. Зачем это нужно

Единый артефакт «хочу купить» для любых товаров витрины. Исполнение (provision и т.п.) подключается отдельно, не в типе файла.

## 3. Как это работает

1. Витрина MARKET собирает `input` по `orderForm` товара (`.product`).
2. Клиент сохраняет `{uid}.bid` через `save_file` на ближайший `$class` категории (например `/MARKET/PAAS`) по роли покупателя. Имя файла — `uid` пользователя, без времени: одна актуальная заявка на пользователя; повторный заказ перезаписывает файл, предыдущие версии — в `history` + `data.logs` (канон §1.6).
3. Тело: `status`, `role: 'USER'`, `buyer`, `created`, `target` (path продукта), `product` (снимок), `input`.
4. Preview показывает JSON заявки.

## 4. Из чего это состоит

- `class.js` — схема FIELDS заявки, label/icon
- `triggers/on_save` — no-op в MVP
- `handlers/preview` — просмотр заявки
