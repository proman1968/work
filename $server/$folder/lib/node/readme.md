# item-node — узел дерева / меню

## Что это

ODA-компонент строки элемента: иконка, подпись и вспомогательные индикаторы. Технически расширяет `item-icon`; прикладное назначение — единый вид узла в дереве, меню и заголовках.

## Зачем это нужно

Один визуальный контракт для любого `$item` в UI: папка, класс, handler, файл. Без отдельного узла дерево и меню не имели бы общей строки с label/icon и контекстными действиями.

## Как это работает

- Рендерит label и icon из `$item`; для handler `file` подставляет расширение из контекста.
- Если у `$item` есть дочерний `readme.md`, показывает кнопку `icons:help`; по нажатию открывает этот файл через `execute` (форма `$md`), не запуская autoRun родителя.
- `readme.md` ищется в `items` / `get_item('readme.md')` — даже если дерево скрыло его из списка.

## Из чего это состоит

- [`node.js`](/$server/$folder/lib/node/node.js/~/handlers/pages/form/) — компонент `item-node`: шаблон, `hasReadme` / `readmeItem` / `openReadme`, label/icon (строка узла и контекстная справка)

## В каком это состоянии

- ✅ отображение label/icon/status
- ✅ кнопка «?» для контекстного `readme.md`
- ✅ открытие readme через `execute` / `window.execute`

## Дальнейшие планы

- При необходимости вынести поиск `readme.md` в общий хелпер с `item-menu`
