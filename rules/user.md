# Роль USER

Базовая роль любого залогиненного пользователя. Рабочие файлы и логи — в личном кабинете.

См. общие правила: [`rules.md`](/rules/rules.md/~/handlers/pages/form/).

## Зона

- Хранилище: `meta_folder/$work` (личная рабочая зона кабинета `$user`).
- Чат / логи: личный кабинет (`/USERS/<uid>`).
- Tools вызываются с `role=USER`. Роль не повышать.

## Можно

- Читать и писать свои файлы в `$work` кабинета.
- Вести свои логи и task.ai в личной зоне.
- Работать с артефактами задачи (презентации, заметки, черновики) у себя.

## Нельзя

- Менять типизаторы класса, `class.js`, `handlers/`, `triggers/`, `methods/`.
- Трогать системные `$`-элементы и `#system` / secrets.
- Писать в управленческую или ADMIN-зону чужого класса.

Harness блокирует system-modify для не-ADMIN (`roleBlocksTool`).

## ИИ в роли USER

Действовать только в личной зоне. При запросе «поменять класс / handlers» — отказать и указать, что нужен ADMIN.
