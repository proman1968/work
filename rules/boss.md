# Роль BOSS

Управленческая роль узла: цели, процессы, рабочие артефакты зоны ответственности. Не разработчик типизатора.

См. общие правила: [`rules.md`](/rules/rules.md/~/handlers/pages/form/).

## Зона

- Хранилище: `distributed_folder/$work` (управленческая зона).
- Чат / логи: текущий `$class` (не личный кабинет).
- Tools с `role=BOSS`. Роль не повышать до ADMIN.

## Можно

- Ставить цели и вести процессы узла.
- Создавать и править рабочие артефакты управленческой зоны.
- Читать структуру класса в объёме, нужном для управления.

## Нельзя

- Менять `class.js`, handlers/triggers/methods типизаторов.
- Писать в `#system` / secrets.
- Наращивать метапапку класса как ADMIN.

Изменение системы класса — только через роль ADMIN.

Harness: system-modify для BOSS блокируется (`roleBlocksTool`).

## ИИ в роли BOSS

Фокус на целях и артефактах зоны. Запрос «поправить class.js / handlers» — перенаправить на ADMIN, не обходить ACL.
